Getting Started with Self-Hosting

2026.05.15 self-hosting

Self-hosting means running your own server — a machine in your home (or a rented VPS) that you control entirely. Your files, your email, your photos, your VPN, your monitoring stack. No terms of service. No company scanning your data to train a model. No subscription that disappears when the startup shuts down.

It is not as hard as it sounds, and the learning curve pays off fast. This is the rough order I'd recommend.

Hardware

You do not need anything special to start. An old laptop, a Raspberry Pi 4 or 5, or any x86 machine with 4GB of RAM and a spare drive will run everything on this page comfortably. My home server is a small-form-factor PC running Ubuntu 24.04 LTS — quiet, low power, and always on.

If you want something cloud-based first, rent a small VPS from Hetzner (cheap, European, good privacy posture) or Linode. A 2-core, 4GB RAM instance is enough to start. You can always migrate later.

Operating System

Ubuntu Server LTS is the easiest starting point — wide documentation, predictable support windows, and everything installs with apt. Debian is a reasonable alternative if you want something more minimal. Avoid anything that requires a GUI; a server running a desktop environment is just wasting RAM and attack surface.

Set up SSH key access immediately after install. Put your public key in ~/.ssh/authorized_keys and disable password login. This is not optional — if your server has a public IP, it will be scanned within minutes of going online.

Docker

Docker is how almost everything runs on my server. Each service lives in its own container — isolated, reproducible, easy to update or roll back. A docker-compose.yml file describes your entire stack: what images to run, what ports to expose, what volumes to mount. Spinning up Pi-hole, Grafana, or Immich is a matter of writing a compose file and running docker compose up -d.

Get comfortable with Docker before adding services. Learn what docker ps, docker logs, and docker compose down do. The rest follows naturally.

What to Run

Start with the things that give you the most back immediately. In rough order of impact:

Pi-hole + Unbound — network-wide DNS blocking and recursive resolution. Every device on your network benefits the moment you point your router's DNS at it. Details in the NetSec guide.

Immich — a self-hosted Google Photos replacement. Facial recognition, albums, shared libraries, a mobile app that auto-uploads in the background.

Jellyfin — self-hosted media server. Stream your film and music library to any device on your network or through a VPN from anywhere.

Vaultwarden — a lightweight, self-hosted Bitwarden-compatible password manager. Your passwords live on your server, not on someone else's cloud.

WireGuard — your own VPN server. Once this is running, everything above is accessible from anywhere as if you were home. Details in the NetSec guide.

Reverse Proxy and SSL

Once you have more than one service running, you want a reverse proxy to sit in front of them — routing pihole.yourdomain.com to one container, jellyfin.yourdomain.com to another, all over HTTPS. Nginx Proxy Manager does this with a web UI and handles Let's Encrypt certificates automatically.

For external access without exposing your home IP, Cloudflare Tunnels are a clean option — a persistent outbound connection from your server to Cloudflare's edge, no port forwarding required.

Backups

Everything on your server should be backed up somewhere off-site. Restic is excellent — encrypted, deduplicated, supports local and remote backends including Backblaze B2 and S3-compatible storage. Run it on a cron job. Test your restores occasionally. A backup you have never tested is a backup you cannot trust.

The 3-2-1 rule: three copies of the data, on two different media types, with one copy off-site. It is the right answer and it has not changed.

Security

A home server exposed to the internet is a real target. The full security stack I run is documented in the NetSec guide — UFW, fail2ban, CrowdSec, WireGuard with Cloak obfuscation, monitoring via Prometheus and Grafana. Do not skip this part. The tools are not complicated and the alternative is a compromised machine on your home network.
