Homelab Full Security Setup

The following categories outline my multi-layered defense strategy. Every service is gated, every log is tracked, and every container is restricted.

SSH Hardening

  • Key-only auth (passwords disabled)
  • Non-standard port (e.g., 2222)
  • AllowUsers whitelist
  • Fail2ban protection (Active)

Firewall (UFW)

  • Default DENY incoming
  • Whitelist-only egress
  • Docker-UFW bypass fix (iptables=false)
  • VLAN isolation (IoT / Personal / Server)

Intrusion Detection

  • CrowdSec: Collaborative IPS
  • Suricata: Deep Packet Inspection
  • Nginx integration for real-time blocking

Monitoring & Alerting

  • Grafana + Loki: Log aggregation
  • Uptime Kuma: Service health checks
  • Netdata: Real-time process monitoring
  • Telegram/Email critical alerts

Container Security

  • Non-root user execution
  • Read-only filesystems
  • Trivy: CVE image scanning
  • Watchtower: Automated updates

SSL & Access Control

  • Nginx Proxy Manager + Let's Encrypt
  • Force HTTPS & HSTS Headers
  • Authelia: 2FA for all services
  • Cloudflare Zero Trust OTP Gate

Audit & Scanning

  • OpenVAS: Scheduled subnet scans
  • auditd: File & process audit logs
  • Forwarding all logs to Loki

Backups & VPN

  • Restic: Encrypted offsite (3-2-1 rule)
  • WireGuard Kill Switch
  • Pi-hole DNSSEC enabled
  • Leak testing via ipleak.net

Resources & Tools

  • CrowdSec — Collaborative security engine.
  • Suricata — Open source IDS/IPS.
  • Trivy — Container vulnerability scanner.
  • Authelia — Single Sign-On & 2FA.
  • Restic — Fast, secure, efficient backup.
  • Uptime Kuma — Self-hosted monitoring tool.
  • Netdata — Real-time performance monitoring.
  • Trivy — Scan your containers for CVEs.
