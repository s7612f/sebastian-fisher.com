Home NetSec Setup

Here is a plain list of the security practices and tools I use to keep my homelab safe.

SSH Hardening

  • Use key-only authentication; disable password logins in sshd_config.
  • Change the default SSH port to something non-standard (e.g., 2222).
  • Implement an AllowUsers whitelist.
  • Run Fail2ban to block brute-force attempts.

Firewall & Networking

  • UFW: Default deny incoming, allow outgoing (whitelist only what you need).
  • Restrict Docker to localhost-only bindings to prevent UFW bypass.
  • Use VLANs to separate IoT devices, servers, and personal devices.
  • Use Pi-hole for network-wide DNS filtering with DNSSEC enabled.

Intrusion Detection & Monitoring

  • CrowdSec: Collaborative IPS to block known bad actors.
  • Suricata: For deep packet inspection.
  • Loki + Grafana: For log aggregation and metrics.
  • Uptime Kuma: Service health checks and alerting.
  • Netdata: Real-time system monitoring.

Container & App Security

  • Run all Docker containers as non-root users.
  • Use read-only filesystems where possible.
  • Trivy: Scan container images for vulnerabilities.
  • Use Watchtower for keeping images updated.

SSL, Access & Backups

  • Use Nginx Proxy Manager for SSL termination with Let's Encrypt.
  • Force HTTPS and enable HSTS headers.
  • Authelia: Add 2FA to services that lack native login.
  • Restic: Encrypted offsite backups using the 3-2-1 rule.
  • WireGuard + ShadowSocks for encrypted, obfuscated VPN traffic.
